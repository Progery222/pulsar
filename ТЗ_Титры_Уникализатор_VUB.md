# Техническое задание: Модуль «Титры» для Уникализатора (VUB) Pulsar

## 1. Введение

Документ описывает добавление в модуль **Video Unique Booster (VUB)** приложения
**Pulsar** новой подсистемы **«Титры»** — генератора точных, стильных, анимированных
текстовых надписей, накладываемых на видео при пакетной обработке.

Цель — дать пользователю профессиональный инструмент титров (вступительные/финальные
заголовки, подписи, субтитры-плашки) с готовыми стилевыми пресетами, точным
позиционированием и анимацией, при этом каждая вариация видео получает **уникальное**
оформление титров (для обхода алгоритмов площадок).

> Подсистема «Титры» **расширяет**, но не заменяет существующую вкладку «Текст» (§4.6
> базового ТЗ VUB). «Текст» — это простая Spintax-надпись; «Титры» — полноценные
> многоблочные стилизованные титры с таймингом и анимацией. На усмотрение реализации
> «Текст» может быть поглощён «Титрами» как частный случай (один блок без анимации).

## 2. Дизайн-система

Интерфейс строго следует палитре Pulsar (§2 базового ТЗ):

| Переменная | HEX | Применение |
|---|---|---|
| `--bg-primary` | `#0D0D0D` | Фон рабочей области |
| `--bg-secondary` | `#1A1A1A` | Фон блоков титров, панелей |
| `--bg-tertiary` | `#252525` | Поля ввода, активные элементы |
| `--accent-green` | `#CCFF00` | Активные элементы, выделение выбранного пресета |
| `--text-primary` | `#FFFFFF` | Основной текст UI |
| `--text-secondary` | `#888888` | Подписи, вторичный текст |
| `--border` | `#2E2E2E` | Границы, разделители |

**Шрифт UI:** Inter (400/600). **Скругления:** 8px (карточки/кнопки), 4px (мелкие элементы).

## 3. Расположение в интерфейсе

Новая вкладка **«Титры»** в левом меню VUB (после вкладки «Текст»), либо переименование
вкладки «Текст» → «Текст и титры». Активная вкладка — текст `--accent-green` + левая
зелёная полоса (как у остальных вкладок).

Центральная область делится на три зоны:

1. **Список блоков титров** (сверху) — вертикальный список добавленных титров.
2. **Редактор выбранного блока** (центр) — все настройки активного титра.
3. **Превью** (справа или снизу) — mockup-кадр 9:16 с отрисовкой титра в реальном времени.

## 4. Модель данных

### 4.1. Блок титра (`TitleBlock`)

Пользователь может добавить **до 10** блоков титров на видео. Каждый блок:

```ts
interface TitleBlock {
  id: string;
  enabled: boolean;            // включён ли блок
  text: string;                // контент, поддерживает Spintax {вар1|вар2}
  preset: TitlePreset;         // стилевой пресет (см. §6)
  // Типографика
  font: FontKey;               // §5
  fontSize: number;            // 12–160 px (в координатах кадра 1080 по ширине)
  fontWeight: 400 | 600 | 700 | 800;
  color: string;               // HEX, основной цвет текста
  letterSpacing: number;       // -5..20 px
  lineHeight: number;          // 0.9..2.0
  uppercase: boolean;          // принудительный верхний регистр
  // Оформление
  stroke: { enabled: boolean; color: string; width: number };       // обводка 0–12 px
  shadow: { enabled: boolean; color: string; blur: number; dx: number; dy: number };
  background: { enabled: boolean; color: string; opacity: number; radius: number; padding: number };
  // Позиция
  anchor: Anchor9;             // 9-зонная привязка (см. §7)
  offsetX: number;             // тонкая подстройка, % ширины кадра (-50..50)
  offsetY: number;             // тонкая подстройка, % высоты кадра (-50..50)
  align: 'left' | 'center' | 'right';
  maxWidth: number;            // макс. ширина блока, % (10..100), для переноса строк
  // Тайминг
  startSec: number;            // время появления, сек от начала
  durationSec: number | 'full';// длительность показа; 'full' = до конца ролика
  // Анимация
  animIn: AnimType;            // появление (см. §8)
  animOut: AnimType;           // исчезновение
  animDurationSec: number;     // 0.1..2.0
}
```

### 4.2. Глобальные настройки титров (`TitlesSettings`)

```ts
interface TitlesSettings {
  blocks: TitleBlock[];
  safeArea: boolean;           // показывать/учитывать safe-зоны TikTok/Reels (см. §7.2)
  variation: TitlesVariation;  // правила уникализации (см. §9)
}
```

## 5. Шрифты

Список шрифтов поставляется с приложением (папка `assets/fonts/`, формат `.ttf`),
чтобы рендеринг был детерминирован и не зависел от системных шрифтов:

| Ключ | Шрифт | Характер |
|---|---|---|
| `inter` | Inter | Нейтральный, по умолчанию |
| `montserrat` | Montserrat | Геометрический, заголовки |
| `oswald` | Oswald | Узкий, «постерный» |
| `bebas` | Bebas Neue | Плакатный, ALL CAPS |
| `roboto` | Roboto | Универсальный |
| `pacifico` | Pacifico | Рукописный акцент |

Шрифты должны корректно поддерживать кириллицу и латиницу. При сборке (`electron-builder`)
папка `assets/fonts` включается в `extraResources`.

## 6. Стилевые пресеты (готовые стильные титры)

Пресет — это преднастроенная комбинация типографики, оформления и анимации в один клик.
Выбирается карточками-превью (мини-рендер «Текст» на тёмном фоне). Активный пресет —
рамка `--accent-green`.

Минимальный набор из **8 пресетов**:

| Пресет | Описание | Ключевые параметры |
|---|---|---|
| **Clean** | Минимализм | Inter 600, белый, без обводки, тонкая тень, fade |
| **Bold Caps** | Крупный заголовок | Bebas/Montserrat 800, uppercase, обводка 4px чёрная, scale-in |
| **Accent** | Брендовый | Inter 700, цвет `#CCFF00`, чёрная плашка 70%, slide-up |
| **Subtitle** | Субтитр-плашка снизу | Inter 600, белый, плашка `#000` 60%, radius 8, fade |
| **Outline** | Контурный | Oswald 700, прозрачная заливка-намёк, обводка 3px белая |
| **Pop** | Динамичный | Montserrat 800, цвет + тень-«дубль», pop (scale+overshoot) |
| **Typewriter** | Печатная машинка | Roboto 600, посимвольное появление |
| **Handwle** | Акцент-подпись | Pacifico, цвет `#CCFF00`, лёгкий наклон, fade |

Выбор пресета заполняет поля блока значениями по умолчанию; пользователь может
дальше править их вручную (пресет → «Свой»/Custom при изменении).

## 7. Позиционирование (точность)

### 7.1. 9-зонная сетка привязки (`Anchor9`)

Сетка 3×3 для быстрой привязки: `top-left | top-center | top-right | mid-left |
center | mid-right | bottom-left | bottom-center | bottom-right`.

Дополнительно — точная подстройка `offsetX` / `offsetY` в **процентах** от размеров
кадра (а не пикселях), чтобы позиция была одинаковой для любого разрешения/формата
(9:16, 1:1, 16:9). Базовые отступы привязки от краёв — **6% по ширине, 8% по высоте**.

### 7.2. Безопасные зоны

При `safeArea = true` в превью отображаются полупрозрачные зоны интерфейса TikTok/Reels
(нижние ~18% — кнопки/подписи, правые ~12% — иконки). Титры, попадающие в эти зоны,
подсвечиваются предупреждением (рамка `--accent-orange`). Авто-сдвиг — опционально.

---

## 8. Анимация (`AnimType`)

Каждый блок имеет анимацию появления (`animIn`) и исчезновения (`animOut`):

| Тип | Эффект |
|---|---|
| `none` | Мгновенно |
| `fade` | Прозрачность 0→1 |
| `slide-up` / `slide-down` / `slide-left` / `slide-right` | Сдвиг + fade |
| `scale` | Масштаб 0.8→1 + fade |
| `pop` | Масштаб 0.6→1.08→1 (overshoot, ease-out-back) |
| `typewriter` | Посимвольное появление (только `animIn`) |
| `blur-in` | Размытие → резкость + fade |

Длительность `animDurationSec` 0.1–2.0 (по умолчанию 0.4). Кривая по умолчанию —
ease-out (cubic). Анимация рендерится через тайм-зависимые выражения (см. §10).

---

## 9. Уникализация титров (вариативность)

Ключевое отличие от обычного редактора: при создании N вариаций видео каждая вариация
получает **слегка отличающиеся** титры, не ломая выбранный стиль. Правила в `TitlesVariation`:

```ts
interface TitlesVariation {
  jitterPosition: { enabled: boolean; rangePct: number };  // дрожание X/Y, ±0..5% (по умолч. 2%)
  jitterSize:     { enabled: boolean; rangePct: number };  // ±0..10% к fontSize
  jitterTiming:   { enabled: boolean; rangeSec: number };  // ±0..0.5с к startSec
  spintax:        boolean;                                  // раскрывать {a|b} по-разному в каждой копии
  colorPool:      string[];                                 // если непусто — цвет текста перебирается по списку
  presetPool:     TitlePreset[];                            // если непусто — пресет чередуется по индексу вариации
}
```

Логика распределения совпадает с уже реализованной для параметров видео:
числовой джиттер распределяется по диапазону по индексу вариации (не случайно рядом),
дискретные наборы (`colorPool`, `presetPool`) **чередуются** по номеру вариации.
См. реализацию [src/vub/ffmpegBuild.ts](src/vub/ffmpegBuild.ts) (функция `value()` —
распределение по полосам, перебор цвета/пресета по `variationIndex`).

## 10. Реализация (FFmpeg)

Рендеринг титров должен быть **независим от логики основного редактора Pulsar**
(как и весь VUB) и встраиваться в общий per-variation пайплайн
[electron/ipc/vub.ts](electron/ipc/vub.ts) → [src/vub/ffmpegBuild.ts](src/vub/ffmpegBuild.ts).

### 10.1. Способ рендеринга

Приоритетный путь — **ASS-субтитры** (формат Advanced SubStation Alpha) через фильтр
`subtitles`/`ass` (libass входит в сборку `ffmpeg-static`). ASS изначально поддерживает:
стили (`Style:`), точное позиционирование (`\pos`, `\an`), обводку (`\bord`), тень
(`\shad`), прозрачность/анимацию (`\fad`, `\t`, `\fscx/\fscy`, `\move`), караоке (`\k`).

Алгоритм на каждую вариацию:
1. Сгенерировать строку `.ass` (в памяти или временный файл): заголовок `[Script Info]`
   с `PlayResX/PlayResY` = разрешение кадра, секция `[V4+ Styles]` со стилями из блоков,
   секция `[Events]` со строками `Dialogue` (тайминг `Start/End`, текст с override-тегами
   анимации).
2. Подключить фильтр: `...,ass='<путь к .ass>'` (или `subtitles=...`). Шрифты —
   через `fontsdir=assets/fonts` и `force_style`/имя шрифта в стиле.
3. Применить **после** базовых видеофильтров (eq/unsharp/grid), но единым `-vf`/filter_complex.

**Маппинг анимаций → ASS override-теги:**

| AnimType | ASS-теги |
|---|---|
| `fade` | `\fad(in_ms,out_ms)` |
| `scale` / `pop` | `\fscx`/`\fscy` + `\t(...)` (для pop — два `\t` с overshoot) |
| `slide-*` | `\move(x1,y1,x2,y2,t1,t2)` |
| `typewriter` | посимвольные `Dialogue` со сдвигом `Start` или `\k`-тайминг |
| `blur-in` | `\blur` + `\t(...)` |

> **Fallback:** если выбран простой режим (один блок без анимации), допускается
> рендер через `drawtext` (как сейчас в `ffmpegBuild.ts`) — `fontfile`, `fontsize`,
> `fontcolor`, `box`, `borderw`, `x/y`. ASS остаётся основным для стилей и анимации.

### 10.2. Spintax и экранирование

`text` сначала проходит раскрытие Spintax (`resolveSpintax`, уже есть в
[src/vub/ffmpegBuild.ts](src/vub/ffmpegBuild.ts)), затем экранирование под выбранный
рендерер (ASS-теги `{`, `}`, `\N` для переноса; либо drawtext-экранирование `:`,`'`,`%`).

### 10.3. Координаты

Все позиции пользователь задаёт в процентах → при генерации `.ass` переводятся в пиксели
относительно фактического `PlayResX/PlayResY` конкретного видео (получать через `ffprobe`,
уже вызывается в `vub.ts`). Это гарантирует одинаковое расположение на любом формате.

## 11. Превью (renderer)

Превью — CSS-отрисовка титра поверх mockup-кадра (как в существующих превью Pulsar),
без вызова FFmpeg: те же значения шрифта/размера/цвета/обводки/тени/позиции в `%`.
Анимации показываются CSS-аналогами (`@keyframes` fade/slide/scale/pop) с тем же таймингом.
Превью отражает **базовый** блок (без джиттера вариаций); опционально — кнопка
«Показать вариацию N» для наглядности уникализации.

---

## 12. UI-компоненты

Переиспользуются существующие компоненты VUB ([src/vub/components/ui.tsx](src/vub/components/ui.tsx)):
`Block`, `Checkbox`, `Switch`, `Select`, `Slider`, `RangeSlider`. Новые элементы:

- **Сетка привязки 3×3** — 9 кнопок-зон, активная `--accent-green`.
- **Карточки пресетов** — горизонтальный скролл мини-превью.
- **Color-поле** — `<input type="color">` в стиле `--bg-tertiary` + HEX-инпут.
- **Таймлайн-полоса** блока (start/duration) — мини-дорожка под списком блоков.

## 13. Значения по умолчанию

- Новый блок: пресет **Clean**, Inter 600, 48px, белый, `anchor = bottom-center`,
  `offsetY = -4%`, `startSec = 0`, `durationSec = 'full'`, `animIn = fade`,
  `animOut = fade`, `animDurationSec = 0.4`.
- `safeArea = true`.
- Вариативность: `jitterPosition = on (2%)`, `jitterSize = off`, `jitterTiming = off`,
  `spintax = on`, `colorPool = []`, `presetPool = []`.

## 14. Acceptance Criteria

1. В VUB присутствует вкладка «Титры»; можно добавить/удалить/включить до 10 блоков.
2. Доступны ≥8 стилевых пресетов; выбор пресета мгновенно меняет превью.
3. Позиция задаётся 9-зонной сеткой + точной подстройкой в %, одинакова для 9:16/1:1/16:9.
4. Поддерживаются обводка, тень, фон-плашка, межбуквенный интервал, uppercase, перенос строк.
5. Анимации появления/исчезновения (минимум fade/slide/scale/pop/typewriter) работают
   в итоговом видео и совпадают по таймингу с превью.
6. Тайминг (start/duration, либо «до конца») корректно применяется.
7. При экспорте N вариаций титры **различаются** согласно правилам `TitlesVariation`
   (джиттер позиции/размера/тайминга, разный Spintax, чередование цвета/пресета),
   не нарушая выбранный стиль.
8. Дизайн вкладки строго соответствует тёмной теме Pulsar с акцентом `#CCFF00`.
9. Логика рендера титров независима от основного редактора и встроена в per-variation
   пайплайн VUB; шрифты поставляются с приложением и работают в собранном `.exe`.
10. Кириллица и латиница отображаются корректно для всех шрифтов.

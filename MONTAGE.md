# Монтаж в Pulsar — полное описание

Автоматический видеоредактор, который **нарезает видео в ритм музыки** (beat-sync).
Идея: загрузил клипы + трек → получил готовый динамичный ролик в один-два клика,
с возможностью ручной докрутки. По духу — авто-режим CapCut.

---

## 1. Поток экранов

```
Главная → Выбор медиа → Выбор музыки → Обработка → Редактор → Экспорт
 home        media         music        processing   editor
```

- **Медиа** — выбираешь видеофайлы (несколько, задаёшь порядок).
- **Музыка** — выбираешь трек (из библиотеки или свой) + сегмент (с какой секунды).
- **Обработка** — анализ ритма + нарезка + расстановка эффектов (автоматически).
- **Редактор** — превью + инструменты + таймлайн.

Файлы: `src/App.tsx`, `src/screens/*`.

---

## 2. Анализ ритма (бит-детект)

- Python-скрипт `python/beat_detect.py` через **librosa** находит **биты**
  (`beat_times`) и **онсеты** (резкие звуки, `onset_times`).
- Результат **кэшируется** по идентичности файла (размер+дата) → один трек считается
  один раз, дальше мгновенно. Кэш: `userData/beat-cache.json`.
- Запуск Python устойчивый: `py -3` → `python` → `python3` (обход зависона Store-алиаса).
- Если Python недоступен — fallback на равномерную сетку.

Файлы: `electron/ipc/audio.ts`, `src/utils/beatDetection.ts`.

---

## 3. Нарезка под биты (Mood)

Точки склейки выбираются по **Mood** (настроение):

| Mood | Когда режет |
|---|---|
| **Mellow** | каждый 4-й бит — спокойно, длинные кадры |
| **Natural** | каждый 2-й бит — сбалансированно |
| **Energetic** | каждый бит + онсеты — максимально динамично |

Клипы берутся из твоих видео **по кругу**, с запретом повторять один и тот же подряд
более 2 раз. Внутри каждого видео «играющая голова» движется дальше, чтобы брать
разные куски.

Файл: `src/utils/videoSlicer.ts`.

---

## 4. Эффекты (12 штук)

**Все 12 реально рендерятся** (не пустышки):

`fastCut · flash · zoom · prism · rgb · boomerang · split · hue · speed · shake · glitch · leak`

- Расставляются **на биты** случайно: уровень 1 ≈ 20% битов, уровень 2 ≈ 40%.
  Минимум 0.5с между однотипными.
- Каждый эффект — короткий импульс (окно ~0.35с), синхронный с битом.
- У каждого есть **вариант** (например zoom: наезд / отъезд / панч) и
  **интенсивность** (0–100).
- В превью эффекты показываются CSS/Canvas в реалтайме, в рендере — теми же
  ffmpeg-фильтрами (логика согласована).

Файлы: `src/data/effects.ts`, `src/data/effectRender.ts` (`buildClipVideoGraph`),
`src/utils/effectsEngine.ts`.

---

## 5. Фильтры-грейды (10 штук)

Глобальный цвет на всё видео:
`warm · cool · vintage · b&w · vcr · glitch · film · light leak · vignette` (+ «нет»).

**Интенсивность 0–100** — плавный блендинг между оригиналом и грейдом
(`blend=all_expr=A*(1-k)+B*k`).

Файл: `src/data/filters.ts`.

---

## 6. Переходы между клипами

Вместо жёстких резов — **xfade** на склейках (~0.25с):

- **Растворение** — мягкий кроссфейд.
- **Слайды** — клипы сдвигают друг друга (влево/вправо/вверх/вниз).
- **Зум / круг** — раскрытие кругом, радиальные.
- **Микс** — случайный переход на каждой склейке.

Аудио тоже кроссфейдится (`acrossfade`). Инструмент **⇄ Переходы**.

Файл: `electron/ipc/ffmpegRender.ts` (этап склейки).

---

## 7. Текст-заголовок

Инструмент **T Текст**: вписываешь заголовок → позиция (верх/центр/низ), цвет,
размер, подложка. Плавное появление/исчезание (drawtext + alpha-рампа).
**Видно на плеере в реальном времени** + в экспорте.

Файлы: `src/components/ToolsPanel.tsx`, `src/components/VideoPreview.tsx`,
`electron/ipc/ffmpegRender.ts` (`buildTitleFilter`).

---

## 8. Стили в один клик

Инструмент **✨ Стиль** — 5 пресетов, каждый настраивает
**ритм + эффекты + переходы + грейд** разом:

- **Чистый** — плавно, растворение, без эффектов.
- **Динамичный** — зум+тряска по битам, микс переходов.
- **Хайп** — быстрые резы, RGB-глитч, VCR-грейд.
- **Эстетика** — тёплая засветка, лёгкий зум.
- **Ретро** — винтаж-плёнка, призма, hue.

Дальше можно докрутить вручную (Mood / Эффекты / Переходы).

---

## 9. Интерактивный таймлайн

Лента клипов под плеером (миниатюры кадров):

- **Перетащить** клип → поменять порядок.
- **Выбрать** клип → кнопка удалить (×).
- **Тянуть зелёные края** → менять старт/длину клипа (trim).
- Тайминги эффектов **пересчитываются автоматически** при любой правке.

Файлы: `src/components/ClipTimeline.tsx`, `src/store/projectStore.ts`
(`reorderClips` / `removeClip` / `trimClip`), `electron/ipc/files.ts`
(`media:thumb` — миниатюры кадров с кэшем).

---

## 10. Аудио

- **Микс**: оригинальный звук видео + музыка, у каждого своя громкость (0–100%).
- **Fade**: in / out / all (0.5с).
- Наложение через `amix`.

---

## 11. Форматы и качество

- **Формат**: 9:16 (TikTok/Reels) · 1:1 (Instagram) · 16:9 (YouTube).
- **Качество**: 720p / 1080p / 4K.
- **Пресеты платформ** (TikTok/Reels/Shorts/YouTube) задают формат + качество + лимит.
- Кодек **H.264 + AAC, 30 fps**.

Файлы: `src/data/platformPresets.ts`, `src/components/ExportModal.tsx`.

---

## 12. FFmpeg-пайплайн экспорта

Файл: `electron/ipc/ffmpegRender.ts` (`renderProject`).

1. **Нарезка фрагментов** — каждый клип масштабируется под формат + накладываются
   эффекты (+ оригинальный звук, если нужен).
2. **Склейка** — concat (жёсткие резы) **или xfade** (если выбраны переходы).
3. **Глобальный грейд** — фильтр с блендингом по интенсивности.
4. **Аудио** — микс музыки и оригинала, fade.
5. **Базовый монтаж** — видео + **текст-заголовок** + fade + аудио.
6. **N уникальных копий** — каждой свои метаданные + смена хэша **валидным
   free-атомом** (TikTok принимает файл, и каждая копия уникальна).

---

## 13. Связка с Уникализатором

Прямо из монтажа можно сделать **несколько уникальных копий** результата
(для разных аккаунтов): каждая получает свои метаданные/хэш — встроено в шаг 6.

> Важно: смена хэша делается **free-атомом** (легальная mp4-подкладка), а не сырыми
> байтами — иначе строгий декодер TikTok отвергает файл («Couldn't decode»).

---

## 14. Инструменты редактора (вкладка Tools)

Лента сверху:
`Videos · ✨Стиль · 🎚Tweak · ⏱Duration · 〜Segment · ☺Mood · ⇄Переходы · T Текст · ▣Fade · ▭Format`

- **Videos** — список и порядок исходных видео.
- **Tweak** — точечная правка одного клипа (выбрать диапазон исходника / заменить клип).
- **Duration** — длина итогового ролика (пресеты или произвольная).
- **Segment** — с какой секунды трека начинать.

---

## 15. Известные ограничения (план доработок)

- Ручные правки таймлайна **сбрасываются** при смене Mood/Duration (там
  пересоздаётся нарезка по битам). → планируется сохранение ручных правок.
- Нет добавления нового клипа на таймлайн вручную (пока только из исходного набора).
- Превью переходов пока не отображается в живом плеере (видно в финальном рендере).
